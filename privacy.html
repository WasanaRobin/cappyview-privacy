Privacy Policy for CappyView

Last Updated: December 11, 2025

Privacy Summary:

CappyView does not collect, store, or transmit any user or patient data. All processing occurs locally in the browser.

This extension stores only non-sensitive user interface preferences on your device. No information ever leaves your computer.

What CappyView Does

CappyView is a Chrome extension that enhances case management workflows by adding navigation tools, customization options, and productivity features to authorized workplace systems.

Data Collection & Storage

Important: CappyView processes case information only within the user's browser. No patient data, personal information, or sensitive content is transmitted, uploaded, or shared. All data remains on your device.

CappyView does NOT collect, transmit, or store:

  • Patient health information (PHI) or medical records
  • Personal identifying information
  • Case data or sensitive business information
  • Usage analytics or telemetry
  • Browsing history or activity outside specified domains
  • Any data sent to external servers

CappyView DOES store locally (on your device only):

  • User interface preferences (sidebar width, toolbar position, theme color)
  • Display settings (PDF zoom level, table filter preferences)
  • Temporary session data for image gallery performance (automatically cleared when gallery closes)

All preference data is stored using Chrome's built-in chrome.storage.local and chrome.storage.sync APIs. This data consists solely of non-sensitive configuration values—UI settings only—and never includes patient information, case details, or any personally identifiable data.

Permissions Explained

CappyView requests the following Chrome permissions. Here's why each is needed:

Storage Permission

Used to save your UI preferences (sidebar width, toolbar state, theme color, PDF zoom level) locally on your device. No patient data or sensitive information is ever stored.

Scripting Permission

Used only on authorized workplace pages to run small, in-page DOM probes via chrome.scripting.executeScript. These probes discover related links (samples, support pages, and attachment PDFs) so the "Open / Close Tabs" tools can correctly identify which tabs belong to the active case. No remote code is executed, no scripts are injected into non-authorized sites, and the results never leave the user's machine.

Tabs Permission

Used to open and manage case-related tabs (Files, Images, Samples, Support pages) based on the current case you're viewing. CappyView does not access tabs outside the specified authorized domains and does not read your browsing history.

Web Navigation Permission

Detects when you navigate between pages in single-page applications to refresh toolbars and sidebars appropriately. Does not intercept, modify, or monitor network traffic.

Host Permissions

CappyView only operates on specific authorized workplace domains that you have legitimate access to as part of your job responsibilities. The extension cannot access any other websites or domains.

Technical Note: This extension is designed for use within authorized organizational systems. Access is controlled by your organization's security policies. The extension operates exclusively on domains you are already authorized to access through your workplace credentials.

Security

CappyView follows Chrome Manifest V3 security standards:

  • All content scripts use strict Content Security Policy
  • DOM insertion is sanitized to prevent XSS attacks
  • Message passing validates origins
  • No eval() or dynamic code execution
  • Service worker background architecture
  • All processing occurs locally—no data leaves your device

For detailed security information, see the Security Documentation.

Your Rights

Since CappyView does not collect or store any personal data:

  • There is no personal data to access, modify, or delete
  • Local preferences can be cleared by uninstalling the extension or resetting Chrome's extension storage
  • No data is shared with third parties
  • All data processing occurs locally on your device
