Privacy Policy for CappyView

Last Updated: December 11, 2025

Privacy Summary

CappyView does not collect, store, or transmit any user or patient data. All processing occurs locally within the user's browser.

This extension stores only non-sensitive user interface preferences on your device. No information ever leaves your computer.

What CappyView Does

CappyView is a Chrome extension that enhances case-related workflows by adding navigation tools, customization options, and productivity features to authorized workplace systems. It operates only within domains the user already has legitimate access to.

Data Collection & Storage

CappyView processes information only within the user's browser. No patient data, personal information, or sensitive content is transmitted, uploaded, or shared.

CappyView does not collect, transmit, or store:

  • Patient health information (PHI)
  • Personal identifying information
  • Case data or sensitive business information
  • Usage analytics or telemetry
  • Browsing history or activity outside specified domains
  • Any data sent to external servers

CappyView does store locally (on your device only):

  • Sidebar width, toolbar position, theme color
  • PDF zoom level
  • Table filter preferences
  • Temporary image gallery session data (cleared automatically)

All local data is stored using Chrome's built-in chrome.storage.local and chrome.storage.sync APIs. These values consist solely of non-sensitive UI configuration settings.

Permissions Explained

Storage Permission

Used to save UI preferences (sidebar width, toolbar state, theme color, PDF zoom level). No patient or sensitive data is ever stored.

Scripting Permission

Used only on authorized workplace pages to run small DOM probes that help identify related internal links (files, images, support pages, attachments). No remote code is executed, and results never leave the user's device.

Tabs Permission

Allows the extension to open and manage case-related tabs. CappyView does not read browsing history and cannot access tabs outside authorized domains.

Web Navigation Permission

Used to detect when the user navigates within single-page applications so the toolbar and sidebar can update. No network traffic is monitored or modified.

Host Permissions

CappyView operates only on specific workplace domains the user already has access to. It cannot access any other websites.

Technical Note: Access to underlying systems is controlled by the organization. CappyView operates solely within the scope of the access the user already has.

Third-Party Code

CappyView includes the following bundled libraries:

  • DOMPurify — Sanitizes HTML for security. Bundled locally, no data transmission.
  • UIcons by Flaticon — Bundled vector icons. Attribution provided per license terms.

No Analytics or Tracking

CappyView includes:

  • ❌ No analytics
  • ❌ No cookies
  • ❌ No advertising
  • ❌ No telemetry
  • ❌ No external requests
  • ❌ No data sharing

All processing is performed locally in the user's browser.

Security

CappyView follows Chrome Manifest V3 security standards:

  • Strict content security policy
  • DOM sanitization to prevent XSS
  • Validated message passing
  • No eval() or dynamic code execution
  • Service worker background architecture
  • Local-only processing (no data transmission)

Your Rights

Since CappyView does not collect personal data:

  • There is no personal data to view, modify, or delete
  • All local preferences can be cleared by uninstalling the extension or resetting Chrome extension storage
  • No data is ever shared with third parties

Updates to This Policy

If CappyView's data practices change, this policy will be updated and the "Last Updated" date will be revised.
